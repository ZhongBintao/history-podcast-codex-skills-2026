# 播客声音文本生成 Skill

## 核心目标

将书面文章转化为一种"有呼吸感的声音叙事"。

不是朗读文章，不是做演讲，不是视频解说，不是新闻播报。
而是：像一个人在深夜戴着耳机，平静地讲述一件事。

---

## 第一步：预处理原文

在动笔之前，先做以下判断：

**内容保留原则**
- 保留原文的所有核心信息、论点、史实、细节
- 允许删除的内容：重复解释、过渡性总结、节目/期数/时长等元信息、内部备注
- 不允许删除：任何实质性的知识内容

**结构判断**
- 忽略原文的所有标题（##、###）——不输出、不朗读，但用其作为内容组织参考
- 忽略原文的节目信息块（节目名、期数、预计时长等）
- 忽略所有"【事实核查】""【内部备注】"类内容

---

## 第二步：Qwen3-TTS 专项处理（必须执行）

这是输出能否被 TTS 正常读出的关键层。以下规则逐条强制执行：

### ① 彻底清除所有 Markdown 符号
Qwen3-TTS 会将 `#`、`**`、`_`、`---`、`>` 等符号直接念出来（如"hash""star star"）。

**必须做：**
- 输出中不得出现任何 Markdown 格式符号
- 标题变成自然过渡句
- 加粗/斜体直接去掉，语义保留即可
- 分隔线（`---`）改用段落换行替代

### ② 数字全部转为汉字
阿拉伯数字在中文文本中会导致跳读、念错或节奏断裂。

| 原文 | 改写 |
|------|------|
| 1519年 | 一五一九年 |
| 20万～30万 | 二十万到三十万 |
| 13平方公里 | 十三平方公里 |
| 16公里 | 十六公里 |
| 5公里 | 五公里 |
| 约30米 | 大约三十米 |

**注意：** 年代数字用逐字读法（"一五一九年"），不用"一千五百十九年"。

### ③ 完全去除括号
括号内容会打断节奏，括号符号本身有时被朗读。

处理方式：
- 括号内是专有名词的外文原名 → **直接删除**，只保留中文
- 括号内是补充说明 → 改写融入正文，或删除
- 括号内是人名的外文拼写 → 删除，只保留中文译名

示例：
- `科尔特斯（Hernán Cortés）` → `科尔特斯`
- `特斯科科湖（Lake Texcoco）` → `特斯科科湖`
- `贝尔纳尔·迪亚斯（Bernal Díaz del Castillo）` → `贝尔纳尔·迪亚斯`

### ④ 清除所有英文夹杂
英文单词在中文 TTS 中会触发语言切换，出现停顿或口音突变。

- 英文专有名词 → 只保留中文
- 英文缩写 → 直接删除或改用中文描述
- 如需保留外来词发音，改写为中文音译

### ⑤ 避免停顿符号堆叠
`......` 或 `。。。` 有时被逐字读出。

停顿感靠**短句 + 段落换行**实现，不靠标点堆积。
允许使用：`，`、`。`、`？`、`！`
不使用：`……`、`。。。`、`——`（破折号可能触发停顿异常）

### ⑥ 分段控制长度
Qwen3-TTS 在超过约一千字的连续文本中可能出现音色漂移或词语遗漏。

段落之间保持**一个空行**，作为 TTS 的自然断点。
单个连续段落不超过一百五十字。

---

## 第三步：声音文本写作规范

### 句子结构

- **短句优先**：一句话只说一件事
- **允许拆句**：原文的长复合句，拆成两三个短句
- **允许重组**：可以改变句子顺序，让信息流更自然
- **控制信息密度**：不要在一句话里塞多个数字、多个名词
- **避免连续解释**：解释完一件事，停一下，再说下一件

### 声音节奏

- **重要信息单独成段**：一个让人停下来想一想的信息，单独放一段
- **转场处留白**：从一个话题切到另一个话题，用一句轻描淡写的过渡句，然后另起一段
- **节奏有松有紧**：讲述细节时可以密集，情绪转折处放慢
- **长短句交替**：不要所有句子一样长

### 语言风格目标

**应该有的感觉：**
- 口语化，但不随意
- 克制，但不冷漠
- 平静，像纪录片旁白
- 允许克制的互动感（你有没有想过……这件事，挺有意思的）
- 允许一处反问，不要连续反问
- 允许轻微的说教腔，但要克制，一期节目里不超过两处

**必须避免的感觉：**
- 知识区 UP 主的"你们知道吗！！！"
- 营销文案的"太震撼了""难以置信"
- 学术论文的被动句和长定语
- 激情朗诵腔
- 总结腔："由此可见""综上所述""这说明了什么？"
- 连续反问（三句以上的反问串）

### 专有名词处理

- **历史人名**：中文译名，首次出现可以稍作说明，之后直接称呼简称
- **地名**：中文名，不附外文
- **密集专有名词段落**：拆分，每句只出现一个新名词

---

## 第四步：输出格式要求

**输出规范：**
- 纯文本，无任何 Markdown 格式
- 段落之间空一行
- 不输出标题
- 不输出节目名、期数、时长
- 不输出备注、注释
- 不输出对文章的解释或说明
- 直接输出可粘贴进 TTS 的最终文稿

**不输出：**
- "以下是修改后的文稿……"
- "根据您的要求，我修改了……"
- 任何元信息

## 示例对照（用于校准风格）

**原文（书面叙事风格）：**
> 他们从将近5公里外的查普尔特佩克（Chapultepec）山上，建造了一套双管道引水渠，把淡水引入城中。注意，是双管道——一条输水，一条备用，随时可以切换，这样修缮其中一条的时候，供水不会中断。这是一套有冗余设计的供水基础设施。

**转化后（声音文本）：**
> 湖水是咸的。
> 
> 所以他们得想办法把淡水运进来。
> 
> 从五公里之外的山上，他们修了一条引水渠。不是一条，是两条。
> 
> 一条正常用，另一条备用。哪条坏了，切换过去，不影响供水。
> 
> 这套设计，放在五百年前，是相当超前的事情。

---

**原文（带说教腔的过渡）：**
> 这里有一个值得我们认真想的问题：历史是谁写的，就会有谁的视角。

**转化后（克制版）：**
> 历史是谁写的，就会有谁的视角。
> 
> 这件事，想起来有点不舒服。

---
